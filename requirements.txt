# Vinkona Knowledge Host dependencies.
#
# The QUERY service + the sqlite backend need NOTHING beyond the Python stdlib
# (FTS5 ships with sqlite3).  numpy makes dense search fast; everything else is
# only needed by the ingestion pipeline, per source type — install what your
# collection contains.  All heavy parsers are lazy-imported, so a missing one
# only disables its own format.

# --- dense search (optional but recommended; falls back to pure-python) ------
numpy

# --- ANN index over node embeddings (optional; `build-ann` + search/reconcile) ---
# HNSW for sub-100ms search past ~1M nodes; absent => exact brute-force matmul.
# Memory-mapped on load, so it does not keep the full matrix resident.
usearch

# --- Wikipedia-scale backend (required for backend = "lance") ----------------
lancedb
pyarrow
pylance     # lance engine; the viewer's random sampling uses its scanner
